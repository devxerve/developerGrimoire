# 💾 Modelo De Datos

## Modelo Conceptual:
Permite representar los datos que se han de almacenar en la base de datos tal como el usuario lo concibe. Estudiaremos el modelo Entidad/Relación. 

## Modelo Lógico:
Menos entendibles, pero traducción directa al modelo físico que entiende el SGBD. Depende del tipo de bases de datos. varios tipos: Jerárquico, **modelo relacional**, orientado a objetos.

## Modelo Físico:
Será distinto según el SGBD a utilizar y se suele expresar en lenguaje tipo SQL.
